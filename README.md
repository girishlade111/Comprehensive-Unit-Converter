Each category has a base unit, and all other units in that category have a conversion factor relative to that base. This makes the logic scalable and easy to update.Conversion Logic: The convert() function is the application's core.For standard units, it converts the input value to the category's base unit and then from the base unit to the target unit.Temperature has a dedicated convertTemperature() function to handle its unique, non-linear conversion formulas.Gemini API Call: The getFunFact() function is an async function that:Constructs a detailed prompt based on the current conversion.Uses the fetch API to make a POST request to the Gemini API endpoint.Parses the response and displays the generated text, with robust error handling and loading states.ğŸ¨ CustomizationAdding a New UnitOpen unit-converter.html and find the units object in the <script> section.Find the desired category and add the new unit with its conversion factor relative to the category's base unit.// Adding Nautical Miles to Length (base: Meters)
